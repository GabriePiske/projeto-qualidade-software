# Diagnóstico de Qualidade – Startup Local Eats

**Equipe:** Tech Quality

**Integrante:** Gabriel Piske

**Sistema:** Local Eats (https://local-eats-unisenac.vercel.app/static/index.html)

---

## 1. Diagnóstico da Situação Atual

### 1.1 Papéis atuais identificados

- Desenvolvedores
- Gerente de projeto

### 1.2 Quem é responsável pela qualidade hoje?

- Atualmente, a qualidade do software provavelmente está sendo responsabilidade dos próprios desenvolvedores, sem um papel específico de QA.

### 1.3 Problemas identificados

- Falta de testes antes da entrega
- Erro ao finalizar pedidos
- Pedidos duplicados
- Funcionalidades com defeitos em produção
- Falta de definição de responsabilidades

### 1.4 Impactos desses problemas

- Esses problemas geram perda de confiança dos usuários, prejuízo para os restaurantes e retrabalho para a equipe de desenvolvimento.

### 1.5 A qualidade é responsabilidade de quem?

- A qualidade deve ser responsabilidade de toda a equipe. Embora exista um papel específico de QA, todos os envolvidos no desenvolvimento devem contribuir para garantir a qualidade do produto.

---

## 2. Papéis da Equipe Propostos

### 2.1 Lista de papéis

- Desenvolvedor
- QA (Analista de Qualidade)
- Analista de Sistemas

### 2.2 Descrição dos papéis

- **Desenvolvedor** → desenvolver funcionalidades, corrigir bugs e realizar testes básicos → garantir que o código funcione com menos defeitos.
- **QA (Analista de Qualidade)** → planejar e executar os testes, registrar bugs e validar funcionalidades → responsável por verificar a qualidade antes da entrega.
- **Analista de Sistemas** → levantar e documentar requisitos → evitar erros de entendimento e garantir que o sistema atenda às necessidades.

---

## 3. Práticas de QA Sugeridas

### 3.1 Lista de práticas

- Testes manuais das funcionalidades
- Registro e acompanhamento de bugs
- Testes exploratórios
- Revisão antes da entrega

### 3.2 Explicação das práticas

- **Prática 01** → Realizar testes manuais nas principais funcionalidades para garantir que estão funcionando corretamente antes da entrega.
- **Prática 02** → Registrar bugs em ferramentas de controle (como Jira ou Trello), permitindo acompanhamento e organização das correções.
- **Prática 03** → Executar testes exploratórios para identificar problemas que não foram previstos inicialmente.

---

## 4. Anúncios de Contratação

### 4.1 Vaga 01 – Analista de Qualidade de Software (QA)

**Descrição da vaga**

A Local Eats busca um profissional de QA para atuar na melhoria da qualidade do sistema, garantindo que as funcionalidades sejam entregues sem erros.

**Responsabilidades**

- Planejar e executar testes
- Identificar e registrar defeitos
- Validar funcionalidades antes da entrega

**Requisitos obrigatórios**

- Conhecimento básico de testes de software
- Boa comunicação
- Capacidade analítica

**Requisitos desejáveis**

- Experiência com testes web
- Conhecimento em automação de testes
- Familiaridade com Git

**Certificações desejáveis**

- ISTQB CTFL

---

### 4.2 Vaga 02 – Desenvolvedor de Software (DEV)

**Descrição da vaga**

A Local Eats procura um desenvolvedor para atuar no desenvolvimento e manutenção da plataforma.

**Responsabilidades**

- Desenvolver funcionalidades
- Corrigir bugs
- Participar de revisões de código

**Requisitos obrigatórios**

- Conhecimento em programação
- Lógica de programação
- Git

**Requisitos desejáveis**

- Experiência com APIs
- Noções de testes automatizados
- Boas práticas de código

**Certificações desejáveis**

- AWS Certified Developer, Oracle Certified Professional (Java), ou similares.

---

## 5. Conclusão da Equipe

A equipe compreendeu a importância da definição de papéis e responsabilidades para garantir a qualidade do software. Uma das principais dificuldades foi identificar corretamente os problemas organizacionais. Como melhoria, foi proposta a inclusão de um papel de QA, definição clara de responsabilidades e adoção de práticas básicas de teste.
